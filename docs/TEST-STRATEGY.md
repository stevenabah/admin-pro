# Admin-Pro 后台管理系统 - 完整测试方案

## 1. 测试计划

### 1.1 测试范围与目标

#### 测试范围

| 模块       | 功能点                          | 测试类型  |
| ---------- | ------------------------------- | --------- |
| 认证与权限 | 登录、登出、Token验证、RBAC权限 | 黑盒+白盒 |
| 用户管理   | CRUD、分页、搜索、状态管理      | 黑盒      |
| 角色权限   | 角色创建/修改/删除、权限分配    | 黑盒      |
| 数据管理   | DataRecord增删改查、状态流转    | 黑盒      |
| 文件管理   | 上传、下载、预览、删除          | 黑盒+灰盒 |
| 媒体管理   | 音视频播放、列表管理            | 黑盒      |
| 首页大屏   | 统计数据、图表渲染              | 黑盒      |

#### 测试目标

- **功能覆盖率**: 核心业务流程 100%，边缘场景 80%+
- **缺陷检出率**: 上线前缺陷检出率 > 95%
- **自动化率**: API测试 100%，E2E测试 > 60%
- **性能指标**: API响应时间 < 500ms，页面加载 < 3s

### 1.2 测试策略

#### 黑盒测试

- 基于需求规格说明书和API文档
- 等价类划分 + 边界值分析
- 场景法覆盖主要业务流程

#### 白盒测试

- 单元测试覆盖率 > 70%
- 关键业务逻辑路径覆盖
- 异常处理路径覆盖

#### 测试环境要求

```
环境配置:
- 前端服务: http://localhost:5173
- 后端服务: http://localhost:3000
- 数据库: SQLite (server/prisma/dev.db)
- Node.js: >= 18.0.0
- npm: >= 9.0.0

默认测试账号:
- 管理员: admin / admin123
- 普通用户: user / user123
```

### 1.3 风险评估

| 风险项           | 概率 | 影响 | 应对措施                                 |
| ---------------- | ---- | ---- | ---------------------------------------- |
| 数据库未初始化   | 中   | 高   | 执行 `npm run db:seed` 初始化数据        |
| 端口冲突         | 低   | 高   | 检查 3000/5173 端口，释放或修改配置      |
| JWT Secret硬编码 | 高   | 中   | 建议改为环境变量（测试环境可忽略）       |
| 第三方服务依赖   | 低   | 中   | 使用Mock服务隔离                         |
| 前端依赖安装失败 | 低   | 中   | 清理缓存后重试 `npm cache clean --force` |

### 3.2 严重程度定义

| 等级 | 定义               | 示例                         | 响应时间 |
| ---- | ------------------ | ---------------------------- | -------- |
| S0   | 系统崩溃，数据丢失 | 登录页面白屏，数据库损坏     | 立即     |
| S1   | 核心功能不可用     | 无法登录，无法创建任务       | 4小时    |
| S2   | 重要功能有缺陷     | 搜索功能异常，权限验证失败   | 24小时   |
| S3   | 一般缺陷           | 样式错位，提示信息不准确     | 72小时   |
| S4   | 体验优化           | 加载动画不流畅，文字轻微错位 | 下版本   |
